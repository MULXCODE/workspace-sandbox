Findex developer screening test

Thank you for your interest in applying to be a developer at Findex. We require all applicants to complete this simple screening test.

Please note that the test is designed to take a small amount of time, and require minimal knowledge of angular. There are no hidden tricks or gotchas, any elegant submission which meets the requirements will result in an interview.

You are welcome to exceed the requirements of the test, and re-write any existing code if you believe it is the right thing to do.

Be sure to read and follow the instructions for submitting your completed test.

Findex Ferry Terminal!

Findex has bought a small ferry terminal and two ferries are available at any one time.

Ferry Types:

Small Ferry
A Small Ferry can hold 8 small vehicles
Large Ferry
A Large Ferry can hold 6 Large vehicles

The Test

There is an API described by the interface IVehicleProvider which currently has one implementation, HardcodedVehicleService, which provides a random vehicle at the click of the button. The goal for this test is to build out the logic to assign these vehicles to the appropriate ferry and to handle when the ferry is full.

The requirements are deliberately left vague to give you the opportunity to show what you can do.

Please note that you are required to submit a solution using angular, other submissions in e.g. react will not be considered. We will take your angular experience (as per your CV/resume) into account when assessing your submission.

Bonus POINTS

Each car type costs a different amount to board a ferry. The Ferry worker earns 10% of the total ticket cost per day.

Car Type Cost
Car $5
Van $7.50
Truck $10
Bus $15

For each vehicle that boards the ferry calculate how much:

  • the terminal earns
  • the worker earns

How the test is assessed

In addition to meeting the requirements, your code will be assessed on the following:

  • Basic UI practices
  • Appropriate use of SOLID programming principles
  • Extensibility of the code
  • Clean code

Submission
